Projet de covoiturage EPF - Documentation des routes

Ceci est une application de covoiturage permettant aux membres de l'EPF de covoiturer pour relier leur trajet domicile-travail.

Endpoints:

Utilisateurs

  • GET /api/users/ - Récupérer tous les utilisateurs.
  • POST /api/user/login - Se connecter en tant qu'utilisateur.
    Paramètres du corps de la requête:
    { "email": "john.doe@email.fr", "password": "admin123" }
  • POST /api/user/logout - Se déconnecter.
    Authentification requise.
  • POST /api/user/register - S'inscrire en tant qu'utilisateur.
    Paramètres du corps de la requête:
    { "first_name": "John", "last_name": "Doe", "email": "john.doe@email.fr", "password": "admin123" }
    Authentification requise pour GET & PUT.
  • GET /api/user/ - Récupérer l'utilisateur authentifié.
    Authentification requise.
  • PUT /api/user/ - Mettre à jour les informations de l'utilisateur.
    Paramètres du corps de la requête (mise à jour de l'utilisateur sans changer le mot de passe):
    { "first_name": "John 2", "last_name": "Doe 2", "email": "john.doe2@email.fr" "password": "admin123" }
    Authentification requise.
  • PATCH /api/user/ - Mettre à jour le mot de passe de l'utilisateur.
    Paramètres du corps de la requête (mise à jour du mot de passe):
    { "email": "john.doe2@email.fr", "password": "admin123", "new_password": "admin1234" }
    Authentification requise.

Courses

  • GET /api/courses/ - Récupérer toutes les courses.
  • GET /api/courses-user/ - Récupérer les courses de l'utilisateur authentifié.
    Authentification requise pour GET & POST.
  • POST /api/courses-user/ - Créer une nouvelle course.
    Paramètres du corps de la requête:
    { "start": "Monoprix Bourg La Reine", "end": "EPF Cachan", "vehicle_brand": "Porsche", "vehicle_model": "Cayenne", "vehicle_seats": 4, "date": "2023-05-15T23:18:32.530856Z", "status": "En attente de passagers" }
  • GET /api/courses/id - Récupérer une course spécifique.
    Authentification requise.
  • PUT /api/courses/id - Mettre à jour une course spécifique.
    Paramètres du corps de la requête:
    { "start": "Monoprix Bourg La Reine", "end": "EPF Cachan", "vehicle_brand": "Porsche", "vehicle_model": "Cayenne", "vehicle_seats": 4, "date": "2023-05-15T23:18:32.530856Z", "status": "Validée" }
    Authentification requise.
  • DELETE /api/courses/id - Supprimer une course spécifique.
    Authentification requise.

Notes

  • GET /api/notes/ - Récupérer toutes les notes.
  • POST /api/notes/id/ - Ajouter une note à une course spécifique.
    Paramètres du corps de la requête (ex. : id de la course 1):
    {}
  • GET /api/notes/courses/course_id/user-rated/rated_id/ - Récupérer, mettre à jour ou supprimer la note donnée par un utilisateur à une course spécifique.
    Authentification requise.
  • PUT /api/notes/courses/course_id/user-rated/rated_id/ - Mettre à jour la note donnée par un utilisateur à une course spécifique.
    Paramètres du corps de la requête:
    {}
    Authentification requise.
  • DELETE /api/notes/courses/course_id/user-rated/rated_id/ - Supprimer la note donnée par un utilisateur à une course spécifique.
    Authentification requise.

Pour plus de détails sur les requêtes, veuillez vous référer au fichier README du projet.
